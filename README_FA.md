# MathFixer — راهنمای فارسی

MathFixer فرمول‌های خراب LaTeX، عبارت‌های UnicodeMath و خطوط معادله‌ای ساده را در فایل Word شناسایی و به فرمول واقعی Microsoft Word (OMML) تبدیل می‌کند؛ بدون اینکه سند از نو ساخته شود.

## اجرای آسان در ویندوز؛ بدون برنامه‌نویسی

از بخش **Releases** فایل `MathFixer-Windows-Portable.zip` را دانلود کنید، آن را
Extract کنید و روی `MathFixer.exe` دوبار کلیک کنید. Python و Pandoc داخل برنامه
قرار دارند و نصب جداگانه لازم نیست. [راهنمای کامل کاربران مبتدی](docs/BEGINNER_GUIDE_FA.md)

## مزیت اصلی

Pandoc فقط خودِ فرمول را به OMML تبدیل می‌کند و هرگز فایل Word را بازسازی نمی‌کند. موتور MathFixer یک نسخه از بستهٔ OOXML اصلی می‌سازد و فقط متن انتخاب‌شدهٔ فرمول را جایگزین می‌کند. فایل اولیه هیچ‌وقت بازنویسی نمی‌شود.

موارد زیر پشتیبانی می‌شوند:

- `$...$`، `$$...$$`، `\(...\)` و `\[...\]`؛
- محیط‌های `matrix`، `cases`، `aligned` و فرمان‌های رایج LaTeX؛
- دلارهای جابه‌جا، براکت‌های ناقص و `\begin{}` بدون نام (با ثبت گزارش ترمیم)؛
- نمادهای UnicodeMath مانند `σ²`، `√x`، `∑`، `≤` و `→`؛
- خطوطی مانند `x = y + 1` که جداکنندهٔ LaTeX ندارند؛
- فرمول‌های داخل جدول، Text Box، سرصفحه/پاصفحه، پانویس، پایان‌نوشت و نظر؛
- پردازش گروهی، Drag & Drop، پیش‌نمایش، حذف موارد اشتباه و ویرایش فرمول ترمیم‌شده؛
- گزارش کامل JSON و کنترل خودکار سلامت خروجی.
- رابط کامل فارسی و انگلیسی؛
- پوسته روشن و تیره؛
- ساخت اختیاری PDF در کنار فایل Word اصلاح‌شده.

## نصب و اجرا

ابتدا Python 3.10 یا بالاتر و Pandoc را نصب کنید:

```bash
python -m venv .venv
# در ویندوز:
.venv\Scripts\activate
pip install -e ".[gui]"
mathfixer doctor
mathfixer-gui
```

برای رابط گرافیکی فقط بستهٔ سبک‌تر `PySide6-Essentials` نصب می‌شود و بستهٔ حجیم و غیرضروری `PySide6-Addons` دانلود نخواهد شد.

## حالت‌های تشخیص

- **Safe:** فقط جداکننده‌های صریح LaTeX؛ کمترین احتمال تشخیص اشتباه.
- **Balanced:** حالت پیشنهادی؛ LaTeX صحیح و خراب، UnicodeMath و خطوط معادله‌ای روشن.
- **Aggressive:** عبارت‌های کوتاه و ریاضی‌محور بدون مرز صریح را هم بررسی می‌کند؛ بهتر است قبل از تبدیل، Preview بازبینی شود.

## ساخت PDF

گزینه «ساخت فایل PDF نیز انجام شود» را فعال کنید. در ویندوز ابتدا Microsoft Word
و سپس LibreOffice امتحان می‌شود. فایل PDF از همان Word اصلاح‌شده ساخته می‌شود و
چیدمان سند دوباره از متن خام بازسازی نخواهد شد.

## تضمین حفظ سند

خروجی فقط زمانی منتشر می‌شود که ساختار ZIP سالم باشد، تعداد جدول‌ها، تصاویر، لینک‌ها، بوکمارک‌ها، کامنت‌ها، Track Changes و Sectionها ثابت مانده باشد، تمام بخش‌های تغییرنکرده بایت‌به‌بایت یکسان باشند و متن اطراف فرمول‌ها هیچ تغییری نکرده باشد.

فرمول‌هایی که فقط به شکل تصویر هستند در نسخهٔ ۱ با OCR تبدیل نمی‌شوند، چون OCR قطعی نیست. تصاویر بدون هیچ تغییری حفظ می‌شوند.

## ساخت فایل EXE مستقل همراه Pandoc

در PowerShell و داخل پوشهٔ پروژه اجرا کنید:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_windows.ps1
```

خروجی در `dist\MathFixer.exe` ساخته می‌شود. Pandoc داخل فایل قرار می‌گیرد و روی
سیستم مقصد به نصب Python، کتابخانه‌ها یا Pandoc نیازی نیست. هنگام انتشار عمومی،
فایل `THIRD_PARTY_NOTICES.md` را نیز کنار برنامه نگه دارید.
